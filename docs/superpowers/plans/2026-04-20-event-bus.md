# Event Bus Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `bus()` as the third effect type in the hex framework, a unified `publish()` verb for both declarative (entity effect) and imperative (workflow / webhook / heartbeat) use, and deprecate `trigger()` with a one-release migration path. Phase 1 scope is the primitive, subscriber wiring, defaults + auto-DLQ, CLI scaffolding, and an integration smoke on `apps/test`.

**Architecture:** New `bus` primitive in `@syncengine/core` with a minimal factory namespace for Layer 1+2 config (`Retention`, `Delivery`, `Storage`, `Retry`, `Backoff`). New `publish()` effect in `emit({ effects })`. `defineWorkflow` accepts `on:` for subscribers and reuses the existing services-injection path. A `BusDispatcher` in `@syncengine/gateway-core` opens one durable JetStream consumer per subscriber, owns delivery-layer retry (NAK-with-backoff, `max_deliver = retry.attempts + 1`), and routes messages to Restate workflow invocations where Restate owns idempotency via invocation id dedup. The imperative `bus.publish(ctx, payload)` resolves its NATS connection from a **per-invocation `AsyncLocalStorage`** set up by the server handler wrapper — no module-level mutable state. `trigger()` remains functional but logs a one-time deprecation warning per process.

**Tech Stack:** TypeScript, Vitest, `@nats-io/jetstream`, `@nats-io/transport-node`, `@syncengine/gateway-core`'s existing `connectNats()`, existing syncengine DSL patterns, Node's `AsyncLocalStorage`.

**Spec:** `docs/superpowers/specs/2026-04-20-event-bus-design.md`

**Out of scope for Phase 1** (Phase 2+):
- Layer 3 `JetStream.*` escape hatch
- `.orderedBy`, `.ordered`, `.concurrency`, `.rate`, `.key` modifiers
- `BusMode.inMemory()` for tests without NATS
- Devtools Buses tab
- Scale-out cross-tier smoke (Phase 3 against `docker-compose.serve.yml`)

## Architectural decisions (locked before task 1)

### Retry ownership

One source of truth: **JetStream owns delivery-layer retry; Restate owns per-invocation idempotency; workflow code owns step retries via `ctx.run`.** Three distinct scopes, no overlap.

- Dispatcher opens the consumer with `max_deliver = retry.attempts + 1` and a `backoff` array derived from `Retry` / `Backoff` config.
- Dispatcher POSTs to Restate ingress with an invocation id derived from `<bus>:<seq>` (or from `.key(fn)` when Phase 2 lands).
- Dispatcher ACKs on Restate 2xx, NAKs-with-delay on Restate 5xx (following the next backoff slot).
- When Restate returns a terminal error (via `TerminalError` in the workflow body), the dispatcher publishes to the DLQ bus and ACKs — no more retries regardless of `max_deliver`.
- The workflow body's own `ctx.run` retries are scoped to individual steps; they do not interact with the dispatcher's retry loop.

### Publisher wiring — AsyncLocalStorage, not a global

`bus.publish(ctx, payload)` resolves its NATS connection from an `AsyncLocalStorage<BusContext>` the server handler wrapper establishes at each invocation entry. This is:

- **Per-invocation**, not per-process — no cross-request contamination in tests or at runtime.
- **Implicit**, so users never pass a connection argument.
- **Co-located** with the existing hex ctx.services injection wrapper — same wrapping point.

The entity runtime does NOT use ALS — it dispatches declared `publish()` effects using the NATS connection it already owns (the same connection the entity runtime uses today for table deltas).

### Bus name regex

`/^[a-z][a-z0-9_-]*$/i` — letters, digits, underscore, hyphen. **No dots.** `.dlq` / `.dead` suffixes are framework-reserved for auto-generated DLQ buses and can't collide with user names.

### Request-id propagation

Every entity handler / workflow / webhook invocation already carries an `x-request-id` (scale-out work landed this). The publish path threads it onto the NATS message headers; the dispatcher pulls it off the redelivered message and forwards it in the Restate invocation headers. One id per logical request, front to back.

```ts
const DURATION_BRAND: unique symbol = Symbol.for('syncengine.duration');

export interface Duration {
    readonly [DURATION_BRAND]: true;
    readonly ms: number;
}

function build(ms: number, source: string): Duration {
    if (!Number.isInteger(ms)) {
        throw new Error(`${source}: count must be an integer`);
    }
    if (ms < 0) {
        throw new Error(`${source}: count must be non-negative`);
    }
    return { [DURATION_BRAND]: true, ms } as Duration;
}

export const milliseconds = (n: number): Duration => build(n, 'milliseconds');
export const seconds = (n: number): Duration => build(n * 1000, 'seconds');
export const minutes = (n: number): Duration => build(n * 60_000, 'minutes');
export const hours = (n: number): Duration => build(n * 3_600_000, 'hours');
```

```ts
import type { NatsConnection } from '@nats-io/transport-node';

export interface BusContext {
    readonly workspaceId: string;
    readonly nc: NatsConnection;
    /** Optional propagated id — set by handler wrappers that received an
     *  x-request-id header. Dispatcher writes it onto outbound NATS headers. */
    readonly requestId?: string;
}

export const busContextStorage = new AsyncLocalStorage<BusContext>();

export function runInBusContext<R>(ctx: BusContext, fn: () => Promise<R>): Promise<R> {
    return busContextStorage.run(ctx, fn);
}
```

```ts
// packages/core/src/bus.ts — attachPublish
function attachPublish<T>(ref: BusRef<T>): BusRef<T> {
    const r = ref as Mutable<BusRef<T>>;
    r.publish = async (ctx, payload) => {
        const parsed = r.$schema.safeParse(payload);
        if (!parsed.success) {
            throw new Error(`bus.publish(${r.$name}): invalid bus payload — ${parsed.error.message}`);
        }
        await ctx.run(`bus:${r.$name}:publish`, async () => {
            // Lazy import to avoid a circular core ↔ server dep at module load.
            const { busContextStorage } = await import('@syncengine/server/bus-context');
            const bc = busContextStorage.getStore();
            if (!bc) {
                throw new Error(`bus.publish(${r.$name}): called outside of a server handler (no bus context)`);
            }
            const subject = `ws.${bc.workspaceId}.bus.${r.$name}`;
            const headers = bc.requestId ? { 'x-request-id': bc.requestId } : undefined;
            await bc.nc.publish(subject, JSON.stringify(parsed.data), headers ? { headers } : undefined);
        });
    };
    return r;
}
```

```ts
// packages/gateway-core/src/bus-backoff.ts
import type { RetryConfig } from '@syncengine/core';

/**
 * Derive the JetStream consumer's `backoff` array from a RetryConfig.
 *
 * JetStream redelivers a NAK'd message after `backoff[i]` nanoseconds
 * where `i` is the redelivery count. We generate one entry per attempt
 * after the first (the first attempt has no wait). `max_deliver` is set
 * to `attempts + 1` (one initial + N retries).
 */
export function retryToBackoffArray(retry: RetryConfig): { backoffNs: bigint[]; maxDeliver: number } {
    switch (retry.kind) {
        case 'none':
            return { backoffNs: [], maxDeliver: 1 };
        case 'fixed':
            return {
                backoffNs: Array.from({ length: retry.attempts }, () => BigInt(retry.interval.ms) * 1_000_000n),
                maxDeliver: retry.attempts + 1,
            };
        case 'exponential': {
            const out: bigint[] = [];
            let delayMs = retry.initial.ms;
            for (let i = 0; i < retry.attempts; i++) {
                out.push(BigInt(Math.min(delayMs, retry.max.ms)) * 1_000_000n);
                delayMs *= 2;
            }
            return { backoffNs: out, maxDeliver: retry.attempts + 1 };
        }
    }
}
```

```ts
// packages/gateway-core/src/bus-dispatcher.ts
import { connectNats } from './nats-connect';
import { retryToBackoffArray } from './bus-backoff';
import { cursorToDeliverPolicy } from './bus-cursor';
import { publishDeadEvent } from './bus-dlq';

export class BusDispatcher {
    private nc: NatsConnection | null = null;
    private stopped = false;
    private activeLoop: Promise<void> | null = null;

    constructor(private readonly config: BusDispatcherConfig) {}

    async start(): Promise<void> {
        this.nc = await connectNats(this.config.natsUrl);
        const js = jetstream(this.nc);
        const streamName = `WS_${this.config.workspaceId.replace(/-/g, '_')}`;
        const { backoffNs, maxDeliver } = retryToBackoffArray(this.config.retry);

        // Retry the consumer-creation for a short window — the stream
        // may not yet exist if the subscriber booted before the
        // workspace was provisioned.
        const consumer = await getConsumerWithRetry(js, streamName, {
            name: `bus:${this.config.busName}:${this.config.subscriberName}`,
            filter_subjects: [`ws.${this.config.workspaceId}.bus.${this.config.busName}`],
            ack_policy: AckPolicy.Explicit,
            max_deliver: maxDeliver,
            backoff: backoffNs,
            ...cursorToDeliverPolicy(this.config.cursor),
        });

        const messages = await consumer.consume();
        this.activeLoop = (async () => {
            for await (const m of messages) {
                if (this.stopped) { m.nak(); break; }
                try {
                    const event = m.json<unknown>();
                    const requestId = m.headers?.get('x-request-id') ?? undefined;

                    if (this.config.filterPredicate && !this.config.filterPredicate(event)) {
                        m.ack();
                        continue;
                    }

                    const invocationId = `${this.config.busName}:${m.seq}`;
                    const result = await postToRestate(
                        this.config.restateUrl,
                        this.config.subscriberName,
                        invocationId,
                        event,
                        requestId,
                    );

                    if (result.kind === 'ok')        { m.ack(); }
                    else if (result.kind === 'terminal') {
                        await publishDeadEvent(this.nc!, this.config.workspaceId, this.config.dlqBusName, {
                            original: event,
                            error: result.error,
                            attempts: m.info.redeliveryCount + 1,
                            firstAttemptAt: Number(m.info.timestampNanos / 1_000_000n),
                            lastAttemptAt: Date.now(),
                            workflow: this.config.subscriberName,
                        }, requestId);
                        m.ack();
                    }
                    else { /* retriable */ m.nak(); }
                } catch (err) {
                    console.warn(`[bus:${this.config.busName}] dispatch error`, err);
                    m.nak();
                }
            }
        })();
    }

    async stop(): Promise<void> {
        this.stopped = true;
        if (this.activeLoop) await this.activeLoop.catch(() => {});
        if (this.nc && !this.nc.isClosed()) await this.nc.drain();
    }
}
```

- [ ] **Step 5: `postToRestate` helper**

Returns `{ kind: 'ok' } | { kind: 'terminal', error } | { kind: 'retriable' }`. Maps Restate's ingress status codes:
- 2xx → ok
- 400 / `application/terminal-error` body → terminal
- 5xx / network → retriable

Restate's invocation-id dedup is what makes this safe — redelivery lands on the same invocation.
